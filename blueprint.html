The Blueprint · Behind the build

How I built an AI
Fractional CMO
in an afternoon.

No developers. No code. Just Claude, a good idea, and 20 years of marketing experience worth distilling.

🪔

The idea

Making myself redundant.
On purpose.

I made myself redundant at Block. Then I did it again — this time deliberately. I wanted to see if I could take 20 years of hard-won marketing experience, distill it into an AI advisor, and put it on my website for anyone to use.

The concept: a fractional CMO that knows my frameworks, my career, my instincts. Sharp, specific, no fluff. To keep it from going completely rogue, I limited it to 3 wishes. Ask your hardest marketing problem. Get the kind of clarity that usually costs $500/hr.

"The wild part wasn't that it worked. It was how fast it worked."

The tools

What I actually used.
All of it, no code.

The whole stack is two tools. That's it.

🤖

Claude (Anthropic)

The AI brain. I used it to write the HTML, design the system prompt, and refine the genie's personality and rules. Claude is the engine under the hood.

🖥️

Cowork (Claude Desktop)

The agentic layer that made it real. Cowork let me direct Claude to build, edit, push to GitHub, and deploy to Netlify — all by talking to it. No terminal. No dev environment.

🐙

GitHub

Version control and the bridge to deployment. Claude pushed every change directly to the repo.

🚀

Netlify

Hosting and serverless functions. Auto-deploys on every push to main. The CMO tool's API calls run through Netlify Functions.

The workflow

Step by step.
Exactly how it happened.

01

Start with the concept, not the code

I described what I wanted to Claude in plain English: a genie lamp metaphor, 3-wish limit, system prompt trained on my actual career. Claude helped me pressure-test the concept before writing a single line.

Claude · Conversation
02

Write the system prompt

This is the most important part. I wrote strict rules: verified facts Claude could reference from my career, what it was never allowed to fabricate, tone, format, and the session wall logic. Claude helped me tighten every line.

Claude · Prompt Engineering
03

Build the page in Cowork

I described the design — dark hero, floating lamp, smoke particles, gold accents — and Cowork built the full HTML/CSS. I reviewed, gave feedback, and it iterated. Back and forth until it looked exactly right.

Cowork · Build
04

Wire up the backend

The tool needed to call the Claude API securely. Cowork built a Netlify Function to handle the API calls server-side, so my API key never touches the browser.

Cowork · Netlify Functions
05

Set up staging and push to live

Cowork cloned my repo, set up a staging branch, and managed every push. Test on staging. Happy? "Promote to live." One sentence and it's done.

Cowork · GitHub · Netlify

Hindsight

What I'd do
differently.

  • Start with the system prompt. I iterated on the design first and had to re-tune the prompt later. Write the AI's brain before you build the body.
  • Set up staging from day one. I pushed a few things directly to live early on. Once I had staging locked, the workflow became clean and stress-free.
  • Nail the session wall sooner. The 3-wish limit and the booking CTA are where the tool converts. I treated it as an afterthought. It should be the first thing you design.
  • Test on real problems earlier. I tested with hypothetical scenarios. Real-world problems exposed gaps in the system prompt I wouldn't have caught otherwise.

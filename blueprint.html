Where it started

I saw someone else do it.
And I wanted mine to feel different.

I'll be honest — I was inspired by a LinkedIn post from someone who had built a fractional CMO using AI. Smart idea. But theirs felt like a bolt-on. A standalone chat window that could have lived anywhere.

I wanted mine to feel like it was actually part of my site — integrated into my brand, my story, my design. Not just a chatbot with my name on it.

So I started building. And the first version was, let's say, functional. It worked. But it was quite plain. The concept was there but the soul wasn't.

"Then I had an idea. What if it was a genie in a bottle? 3 wishes. A lamp. Smoke. Magic."

That was mine. Claude didn't suggest it. I did. And then Claude took that idea and made it something I'm genuinely proud of — the floating lamp, the smoke particles, the gold glow, the wish counter. The execution was a collaboration. The concept was my brain.

How it actually went

The messy reality
of building with AI.

I'm not going to pretend this was a smooth 5-step process. It wasn't. Here are the honest moments from the build:

The hallucination problem

Early on, the genie started making up specific stories from my career — invented numbers, fabricated campaign results. Things I never said or did. I had to hard-code strict rules: the genie can only reference verified facts about my career, nothing else. That took real work to get right.

The API key saga

Getting the Anthropic API connected was... a journey. Missing environment variables. A truncated key that looked right but wasn't. A credit balance that was there but not registering. Each issue looked different to the last. Eventually resolved, but it took longer than I'd like to admit.

The dead-end form

The first version had a form that just... stopped. You'd fill in your challenge, hit submit, and get a "session is live" message. Then you had to type your question again. A complete dead end. Fixed it so your challenge fires immediately as the first message.

The Netlify quirk

Netlify injects scripts that can corrupt inline JavaScript. Spent time debugging functions that just weren't being found. Fix: everything has to be explicitly assigned to the window object. One of those things you only learn the hard way.

Moving from chat to Cowork

I built the early version by manually downloading files from Claude chat and uploading them to GitHub and Netlify one by one. Every. Single. Change. Moving to Cowork changed everything — it manages the files, the git workflow, staging vs live. Night and day.

The actual stack

What I used.
All of it.

No developers. No code written by hand. Two AI tools and a bit of persistence.

🤖

Claude (Anthropic)

The brain. Built the HTML, CSS, designed the system prompt, refined the genie's personality and rules.

🖥️

Cowork

The hands. Manages files, pushes to GitHub, deploys to Netlify. You just talk to it. No terminal required.

🐙

GitHub

Version control. Staging branch for testing, main branch for live. Cowork handles all the pushing.

🚀

Netlify

Hosting + serverless functions. The API key never touches the browser — it all runs server-side.

Hindsight

What I'd do
differently.

  • Start with Cowork, not chat. The manual download-upload loop in early Claude chat was painful. Cowork handles all of that automatically. Start there.
  • Write the system prompt first. I built the design before nailing the AI's rules. The genie's brain should come before its face.
  • Tackle the hallucination rules on day one. It's uncomfortable to find your AI making up stories about your career. Lock the verified facts list down immediately.
  • Design the session wall early. The 3-wish limit and booking CTA is where the tool actually converts. I treated it as an afterthought. It's the most important bit.
  • Set up staging before touching live. A few early pushes went straight to production before I had staging sorted. Every change should go to staging first.
